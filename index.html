About this app

This calculator application is complete with different types of Switch Controls. The first is 5 switch control using the arrow keys to navigate and the enter button to select. The application also has 3 switch control where the user can navigate to previous and next buttons using the left and right shift buttons, and select using the space bar. The user could use a combination of the arrow keys & shift keys & enter/space bar to use the calculator, but for the purposes of testing the 3 key vs 5 key control application, the space bar will be used with the shift keys and the enter key will be used with the arrow keys. There is also the option to use the mouse to click on the large navigation buttons to the right of the calculator.

Instructions

One way you can navigate the calculator is by using the large buttons shown on the screen to account for slightly inaccurate mouse movements. Alternatively, you can navigate the calculator using the arrow keys on your keyboard and the enter button as the select button (5 switch control). In case the arrow keys are too close together (causing inaccurate button selection) the application also has 3 switch control, where the left shift corresponds to the LEFT button (Select Previous) and the right shift corresponds to the RIGHT botton (Select Next) and the space bar corresponds to the select button.

Test Description / Results

I tested my application by comparing the two different versions of my interface (3-key and 5-key). I completed two trials of five different equations (shown below) and recorded the time it took to complete the equations using the two different methods. After completing the testing, I found that I was more accurate in navigating the calculator when using the 3 switch control (consisting of the left and right shift keys and the space bar). I only made one unintended key press because I accidentally hit the select button twice. However, given that theys keys were more spaced out, it was nearly impossible to not select the intended button, though the 3 key method was notably slower. When using the 5 switch control (the arrow keys for navigation and the enter key for select), I found that it was more likely to select the unintended arrow key because of their close proximity. For someone who may have to use a fist or larger object to nagivate the calculator, it would be difficult to select the arrow keys accurately. However, the 5 switch control did prove to be much faster than 3 switch control as I could quickly complete the equations using my index finger. I believe that the three button control would allow someone to accurately use their fist/large object to navigate the calculator, eventhough it would be much slower since it only has "Select Next" and "Select Previous" functionality. After completing my test, the data demonstrated that using the 5 key control was faster, but using the 3 key control was more accurate. However, depending on the limitations of the user, one of the methods may be preferable to the other.
